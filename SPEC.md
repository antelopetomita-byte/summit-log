- **リポジトリ**: https://github.com/antelopetomita-byte/summit-log

## 1. 概要

世界中の山への登山記録(日付・山名・行程・感想)を日々書き溜め、最終的に**書籍原稿として書き出す**ことを目的とした個人用ジャーナルアプリ。iPhoneのSafariで動作し、ホーム画面に追加してアプリのように利用する。

### 目的
1. 遠征中でも1分で書ける、続けやすい記録体験
2. 記録がそのまま本の原稿になる書き出し機能
3. サーバー不要・アカウント不要・完全無料での運用

## 2. 技術構成

| 項目 | 内容 |
|---|---|
| 形式 | 単一HTMLファイル(index.html)、依存ライブラリなし |
| ホスティング | GitHub Pages |
| データ保存 | ブラウザのlocalStorage(キー: `summitlog.v1`) |
| フォント | Google Fonts: Shippori Mincho(見出し)/ Zen Kaku Gothic New(本文) |
| 対応環境 | iOS Safari 15+、その他モダンブラウザ |
| 通信 | フォント読込のみ。記録データは一切外部送信しない |

## 3. 画面構成

下部タブバーで3画面を切り替える。

### 3-1. 記録一覧(初期画面)
- ヘッダー統計: **登頂記録数 / 累計標高(m) / 国・地域数** を自動集計
- 縦のタイムライン表示(新しい順)。各カードに日付・天候・山名・国・標高・本文冒頭2行
- テキスト検索: 山名・国・本文・行程を部分一致でフィルタ
- カードタップで詳細モーダル(全文表示+編集ボタン)

### 3-2. 記録する(入力フォーム)
| 項目 | 型 | 必須 |
|---|---|---|
| 日付 | date(初期値: 当日) | ✔ |
| 山名 | text | ✔ |
| 国・地域 | text | |
| 標高(m) | number | |
| 天候 | text | |
| 同行者 | text | |
| ルート・行程 | text | |
| 記録・感想 | 長文textarea | |

- 保存時にバリデーション(日付・山名必須)
- 既存記録の編集モードでは削除ボタンを表示(confirm付き)

### 3-3. 本にする(書き出し)
| 機能 | 出力 | 内容 |
|---|---|---|
| 書籍原稿(Markdown) | `summit-log_YYYY-MM-DD.md` | 下記4章立てロジック参照 |
| プレーンテキスト | `summit-log_YYYY-MM-DD.txt` | 日付順の全文 |
| クリップボードコピー | — | Markdown原稿をコピー |
| バックアップ保存 | `summit-log_backup_YYYY-MM-DD.json` | 全データのJSON |
| バックアップ復元 | JSON読込 | ID重複を除外して統合(上書きしない) |

iOSでは共有シート(`navigator.share`)経由でファイル保存。非対応環境ではダウンロードリンクにフォールバック。

## 4. 書籍原稿の章立てロジック

1. 全記録を**日付昇順**に整列
2. 扉ページ: タイトル「世界の山を歩く」+ 著者「中村紳一郎 遠征記録 — 全n篇」
3. 山名が切り替わるたびに `## 山名(国 / 標高m)` の章見出しを挿入
4. 各記録は `### YYYY年M月D日` の節見出し+メタ情報(天候/同行/行程を引用ブロック)+本文

同じ山に複数日の記録がある場合、1つの章にまとまる(連続している場合)。KDPや出版社持ち込みの下書きにそのまま利用可能。

## 5. データモデル

```json
{
  "id": "英数字ユニークID",
  "date": "YYYY-MM-DD",
  "mt": "山名",
  "country": "国・地域",
  "elev": 4808,
  "wx": "天候",
  "party": "同行者",
  "route": "ルート・行程",
  "body": "本文"
}
```

localStorageに上記オブジェクトの配列をJSON文字列で保存。

## 6. デザイントークン

| トークン | 値 | 意味 |
|---|---|---|
| `--ink` | `#0C1220` | 背景(夜の山) |
| `--surface` | `#121C30` | カード面 |
| `--glacier` | `#7FD8E8` | アクセント(氷河) |
| `--lamp` | `#E8A852` | タイムライン点(ヘッドランプ) |
| `--danger` | `#E87070` | 削除系 |

シグネチャ要素: 破線の縦タイムライン+橙のグロー点(夜間登山のヘッドランプの列がモチーフ)。

## 7. 制約事項・運用上の注意

- データは**端末のブラウザ内のみ**に保存される。以下の場合に消失リスクがある:
  - Safariの「履歴とWebサイトデータを消去」
  - 長期間(目安7日以上)サイトを開かない場合のiOSによるストレージ削除
  - 機種変更
- **対策**: ホーム画面に追加した状態で定期的に開くこと。遠征前・帰国後にJSONバックアップを保存する運用を推奨
- 写真の添付は非対応(v1)。localStorage容量(約5MB)の制約のため
- 複数端末間の同期は非対応
