## 1. 概要

世界中の山への登山記録(日付・山名・行程・感想)を日々書き溜め、最終的に**書籍原稿として書き出す**ことを目的とした個人用ジャーナルアプリ。iPhoneのSafariで動作し、ホーム画面に追加してアプリのように利用する。

### 目的
1. 遠征中でも1分で書ける、続けやすい記録体験
2. 記録がそのまま本の原稿になる書き出し機能
3. サーバー不要・アカウント不要・完全無料での運用

## 2. 技術構成

| 項目 | 内容 |
|---|---|
| 形式 | index.html + manifest.json + sw.js + アイコン2点 |
| ホスティング | GitHub Pages |

| 対応環境 | iOS Safari 15+、その他モダンブラウザ |

## 3. 画面構成

下部タブバーで4画面を切り替える。

### 3-1. 記録一覧(初期画面)
- ヘッダー統計: **登頂記録数 / 累計標高(m) / 国・地域数** を自動集計
- 縦のタイムライン表示(新しい順)。各カードに日付・天候・山名・国・標高・本文冒頭2行
- テキスト検索: 山名・国・本文・行程を部分一致でフィルタ
- カードタップで詳細モーダル(全文表示+編集ボタン)

### 3-2. 記録する(入力フォーム)
| 項目 | 型 | 必須 |
|---|---|---|
| 日付 | date(初期値: 当日) | ✔ |
| 山名 | text | ✔ |
| 国・地域 | text | |
| 標高(m) | number | |
| 天候 | text | |
| 同行者 | text | |
| ルート・行程 | text | |
| 緯度・経度 | number(「山名から検索」ボタンでNominatim自動入力可) | |
| 写真 | 最大3枚。長辺1280px・JPEG品質0.8に自動圧縮しIndexedDBへ保存 | |
| 記録・感想 | 長文textarea | |

- 保存時にバリデーション(日付・山名必須)
- 既存記録の編集モードでは削除ボタンを表示(confirm付き)

### 3-3. 登頂マップ
- 緯度経度が入力された記録を世界地図(OpenStreetMap)上に橙のピンで表示
- ピンのポップアップから該当記録の詳細を開ける
- 記録がある場合は全ピンが収まるよう自動でズーム調整

### 3-4. 本にする(書き出し)
| 機能 | 出力 | 内容 |
|---|---|---|
| 書籍原稿(Markdown) | `summit-log_YYYY-MM-DD.md` | 下記4章立てロジック参照 |
| プレーンテキスト | `summit-log_YYYY-MM-DD.txt` | 日付順の全文 |
| 縦書きPDF | 印刷ダイアログ | 縦書き組版(vertical-rl)の書籍レイアウトを生成し、iOSの「プリント→PDFとして保存」でPDF化 |
| クリップボードコピー | — | Markdown原稿をコピー |
| バックアップ保存 | `summit-log_backup_YYYY-MM-DD.json` | 記録+**写真を含む**全データ(`{version, entries, photos}`形式。v1の配列形式も復元可) |
| バックアップ復元 | JSON読込 | ID重複を除外して統合(上書きしない) |

iOSでは共有シート(`navigator.share`)経由でファイル保存。非対応環境ではダウンロードリンクにフォールバック。

## 4. 書籍原稿の章立てロジック

1. 全記録を**日付昇順**に整列
2. 扉ページ: タイトル「世界の山を歩く」+ 著者「中村紳一郎 遠征記録 — 全n篇」
3. 山名が切り替わるたびに `## 山名(国 / 標高m)` の章見出しを挿入
4. 各記録は `### YYYY年M月D日` の節見出し+メタ情報(天候/同行/行程を引用ブロック)+本文

同じ山に複数日の記録がある場合、1つの章にまとまる(連続している場合)。KDPや出版社持ち込みの下書きにそのまま利用可能。

## 5. データモデル

```json
{
  "id": "英数字ユニークID",
  "date": "YYYY-MM-DD",
  "mt": "山名",
  "country": "国・地域",
  "elev": 4808,
  "wx": "天候",
  "party": "同行者",
  "route": "ルート・行程",
  "lat": 45.83262,
  "lng": 6.86444,
  "photos": ["写真ID", "..."],
  "body": "本文"
}
```

記録本体はlocalStorageに配列で保存。写真は `{id, data(JPEG dataURL)}` としてIndexedDBに保存し、記録側はIDのみ参照する。

## 6. デザイントークン

| トークン | 値 | 意味 |
|---|---|---|
| `--ink` | `#0C1220` | 背景(夜の山) |
| `--surface` | `#121C30` | カード面 |
| `--glacier` | `#7FD8E8` | アクセント(氷河) |
| `--lamp` | `#E8A852` | タイムライン点(ヘッドランプ) |
| `--danger` | `#E87070` | 削除系 |

シグネチャ要素: 破線の縦タイムライン+橙のグロー点(夜間登山のヘッドランプの列がモチーフ)。

## 7. 制約事項・運用上の注意

- データは**端末のブラウザ内のみ**に保存される。Safariのデータ消去・長期未使用・機種変更で消失リスクあり
- **対策**: ホーム画面に追加(PWAインストール)した状態で使うこと。遠征前・帰国後に完全バックアップ(写真込みJSON)を保存する運用を推奨
- 写真はIndexedDB保存のため実用上数百枚まで対応。ただしバックアップJSONは写真枚数に比例して大きくなる
- 位置検索(Nominatim)と地図タイルはオンライン時のみ。一度表示した地図範囲はキャッシュされオフラインでも閲覧可
- 複数端末間の同期は非対応
